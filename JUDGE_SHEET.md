# Richmond Civic Hackathon — Judge Sheet

## What you're doing

Evaluate each project using the same 6 categories.
Score each from **1–5**, then apply the **weights for the award type**.

---

# Scoring Categories (1–5)

### 1. Impact (MAP Alignment)

Does this meaningfully advance the problem or the targeted pillar?

* 1 = weak / unclear
* 3 = solid alignment
* 5 = strong, measurable civic impact

---

### 2. Feasibility

Could this realistically be piloted or implemented?

* 1 = unrealistic
* 3 = plausible with effort
* 5 = ready for near-term pilot

---

### 3. User Value

Does this create real value for a clear user?

* 1 = unclear user / weak value
* 3 = useful
* 5 = highly valuable, solves real pain

---

### 4. Execution

How well was it built and demonstrated?

* 1 = mostly conceptual
* 3 = functional
* 5 = strong, clear working demo

---

### 5. Innovation

Is the idea original or insightful?

* 1 = generic
* 3 = some originality
* 5 = highly novel or insightful

---

### 6. Equity & Inclusion

Does it improve access or reduce barriers?

* 1 = ignored
* 3 = considered
* 5 = meaningfully improves equity

---

# Final Score

```
Final Score = Σ (category score × weight)
```

---

# Judge Guardrails

* Do NOT reward complexity over usefulness
* Do NOT reward polish over real impact
* Penalize unrealistic assumptions (data, integration, policy)
* Anchor decisions in **public value**, not "cool factor"

---

# Quick Mental Check

* **Mayor's Choice:** Would the City pilot this soon?
* **Moonshot:** Could this change how the city operates over time?
* **Pillar:** Is this the best answer to the targeted pillar?
