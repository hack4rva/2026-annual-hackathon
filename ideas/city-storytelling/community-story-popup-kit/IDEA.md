# Community Story Popup Kit

**Source:** NEW  
**Pillar:** A City That Tells Its Stories  
**Problem Statement:** Give residents a trusted, accessible way to share their lived experiences so community narratives can inform City decisions.

---

## Description

The kit is not the innovation — the **pipeline** is. Anyone can buy a Zoom H5; the value is a standardized, repeatable workflow: capture → consent → backup → transcribe → moderate → archive → reuse, simple enough that a library volunteer can run it. Richmond lacks a standardized, low-cost, repeatable toolkit for capturing resident stories at public events, so story collection is ad hoc, inconsistent, and disconnected from any preservation pipeline. The critical reframe: make the pipeline so simple that community events can feed Richmond Stories Online or VCU Libraries through defined handoff points (e.g., library-lendable “Richmond Story Kit,” festival quick-capture, popup-to-archive).

## Key Approach

- Lendable physical kit (recorder, mics, consent forms, prompts, backup instructions) via Richmond Public Library, modeled on other libraries’ oral-history lending programs.
- Event workflow: reserve/check out → use at event → upload audio to secure folder → return kit → stories enter moderation and publication.
- Popup-to-archive pipeline: field capture, backup, automated transcription (e.g., Otter.ai), human review, metadata, deposit to Omeka S or a public feed.
- Consent and access: default to non-exclusive licenses so residents retain copyright; include multilingual prompts where possible.
- Optional city-facing service: staffed quick-capture stations at major festivals and cultural events for higher throughput.
