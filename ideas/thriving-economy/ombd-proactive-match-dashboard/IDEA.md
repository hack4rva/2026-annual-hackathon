# OMBD Proactive Match Dashboard

**Source:** NEW  
**Pillar:** A Thriving Economy  
**Problem Statement:** Help minority-owned businesses identify and understand City contracting opportunities that match their capabilities.

---

## Description

**Core assumption challenged:** Richmond’s OMBD needs a better vendor directory; the analysis argues the gap is **proactive matching** between certified vendors and upcoming opportunities — not replacing B2Gnow but surfacing what passive browsing misses. Certified MBEs cannot efficiently discover which upcoming City contracts match their capabilities when internal systems are not vendor-facing and alerts are noisy. The idea is a **public-facing match dashboard** (or feed) that shows which upcoming contracts best match a vendor’s profile, with match rationale — created as a layer that can use public contract data and self-reported vendor profiles where API access to internal directories is unavailable.

## Key Approach

- Vendor profile (e.g., NAICS/NIGP, capabilities) → ranked list of upcoming or active contracts from public sources (e.g., Socrata City contracts), with transparency on why items matched.
- Reduce alert fatigue by favoring precise commodity coding over overly broad categories, per research cited in the analysis.
- Staff-facing view: generate curated shortlists for outreach to qualified MBEs for specific solicitations.
- Optional stretch: utilization or accountability views if data supports MBE spend reporting over time.
- Validate early: usefulness of Socrata-only matching without B2Gnow API, vendor willingness to maintain profiles, and whether the dataset supports forward-looking, not only historical, matching.
