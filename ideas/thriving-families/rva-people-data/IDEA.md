# RVA People Data — Unified Public Health & Equity Data Portal

**Pillar:** Pillar 3 — Thriving Families, Problem 2
**Origin Quote:** Jackie Hale at the kickoff session said three words: *"people data portal"* — and added *"she wants it for a job so bad"*
**Data:** 173 verified indicators from 7 official sources — race-disaggregated by default, Virginia-benchmarked, export-ready
**Automation:** GitHub Actions workflow for annual data refresh
**Hackathon:** Hack for RVA

---

## Problem Statement

Every maternal health organization, every city department, every nonprofit — they all download the **same five government data sets** (VDH, Census, VDOE, etc.) in different formats, then show up to meetings with **different metrics**. The result:

- Nobody is working from the same denominator
- Grant packs get rebuilt from scratch every funding cycle
- Equity gaps sit buried in spreadsheets nobody has time to dig through
- Budget decisions get made on vibes

---

## Solution

**RVA People Data** — 173 verified indicators, 7 official sources, all in one place. Race-disaggregated by default. Virginia-benchmarked. Export-ready.

A maternal health analyst can open this, find Richmond's preterm birth rate (**12.4%**, next to the Virginia average), with the source cited, and be done in **3 minutes instead of 3 hours**.

---

## Data Sources

7 official data sources are unified in the portal, including:
- **VDH** (Virginia Department of Health)
- **Census / ACS** (American Community Survey)
- **VDOE** (Virginia Department of Education)
- **CDC Wonders**
- **CDC Places**

---

## Interface Structure

The portal is organized into **4 main tabs**:
1. **Health**
2. **Population**
3. **Poverty**
4. **Education**

Each tab contains subdivisions — for example, under Health:
- Adolescent health
- Birth outcomes
- HIV
- Infant mortality
- And more

Each subdivision has its own:
- Data sets
- Charts/graphs
- Data forms

---

## Feature: Equity/Disparity Snapshot

A dedicated **equity snapshot view** provides an instant visual of racial disparities across key indicators — showing the difference between racial groups in the community at a glance. This is designed to make systemic gaps immediately visible, not buried in tables.

---

## Feature: Custom Chart Builder

Users can describe a chart they want to see, and the system builds it dynamically from the portal's internal data:

- Example: *"Compare poverty rate and population of the Black community"*
- If the exact indicators don't exist, the system says so — it **does not hallucinate** and **does not web search** for data; everything is sourced from within the portal
- Rate limits: **5 custom graphs per hour per session**, resetting after one hour
- All charts include the **data source** and the **time it was generated**

---

## Feature: Data Packs (Free Download)

Two tiers of downloadable data packs, both **completely free**:

### Standard
- Downloads data from a specific domain the user selects
- Example: select "Health" → download all available health data

### Complete
- Downloads the entirety of the database
- Customizable — users can specify exactly which data they want to include

---

## Feature: MCP Connector (AI Integration)

From the About section, users can **connect the data portal to their AI tools via an MCP connector**:

1. Copy the provided connector link
2. Go to connectors in the target AI tool
3. Add the connection

The presenter demonstrates this live — 4 custom tools are exposed through the connector, allowing an AI assistant to search through the portal's data directly. The entire search history shown is generated by the AI calling through those tools in real time.

---

## Feature: CSV Upload Pipeline

Staff or administrators can upload new CSV datasets to add them as public indicators:

1. Log into the **admin portal** (password-protected authentication)
2. Browse to a CSV file
3. **Map the CSV fields** to the appropriate indicator name
4. Submit — the data is added as a new metric in the system immediately

The presenter notes that the current authentication is simple/basic and could be strengthened. The pipeline can be extended to support Tableau images or PDFs in the future.

---

## Automation: Annual Data Refresh

The portal has a **GitHub Actions workflow** that runs on a yearly basis and automatically:
- Re-calls the Census ACS API
- Re-calls CDC Wonders
- Re-calls CDC Places
- Updates all data sets automatically

This means the portal stays current without manual intervention year over year.

---

## Design Philosophy

The system is explicitly designed to **not hallucinate** — it does not pull external data or web search for answers. Every result shown to users comes only from data already loaded into the portal. This makes the portal a trusted, auditable source rather than a generative one.
