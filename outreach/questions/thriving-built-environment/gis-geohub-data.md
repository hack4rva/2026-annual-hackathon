# Outreach: Richmond GIS / GeoHub Data Manager — Research & Questions

I'm [your name] from the RVA Hacks civic hackathon organizing team. At our March 2026 event, a team of three J.R. Tuggle High School juniors built a remarkable product called StreetLights — a plain-language, resident-facing infrastructure project map that pulls live data from Richmond GeoHub. The site is live at https://ang-aq.github.io/Streetlights/.

The core idea is exactly right: take GeoHub's real project data and make it legible to ordinary residents. We think this could be genuinely useful for Richmond. But its usefulness depends entirely on GeoHub data being current and accurate — and we don't know the answer to that question.

I'd love to ask you a few quick questions about how GeoHub data is maintained, so we can give the team an honest assessment of what would need to change for this to be a reliable resident-facing tool.

**Our questions:**
1. What is the typical update cadence for active infrastructure project records in GeoHub? Do project managers update their own records, or is there a central data team?
2. Does GeoHub include projects in all phases — planning, design, and active construction — or primarily construction-phase projects?
3. Do GeoHub project records typically include project manager contact information? (StreetLights displays this, and it would be a valuable resident feature if the data is reliable.)
4. Does GeoHub include VDOT-managed roads (state highways) as well as City-managed roads, or only City projects?
5. Is there a mechanism for residents or external tools to flag stale or incorrect project data back to the City?
6. Would the City be open to a formal data quality agreement with a tool like StreetLights — essentially committing to keep project records current in exchange for a public-facing tool that reduces resident 311 calls about construction?

We're not asking for anything proprietary. Just trying to understand the data ecosystem so we can help the team understand the path from prototype to reliable product.

## Context for the Reviewer of This File
StreetLights is the most promising demo from the Built Environment pillar because it uses real GeoHub data — unlike the fleet demos which used synthetic data. The team built a solid, working prototype. The product's reliability as a resident-facing tool depends on whether GeoHub records are kept current by project managers. If they're not, StreetLights shows residents stale or wrong information, which is worse than no information. This outreach is about understanding the governance reality before advising the team to invest more in the product.

## Why This Contact
The GIS Manager or central data team member is the right person because they understand GeoHub's data governance model — whether project managers self-update, whether there's quality review, and whether a formal data partnership is feasible.
