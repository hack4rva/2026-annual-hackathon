## Seven personas (condensed)

### 1 — MAYOR (Decider)

**Advance / Refine:** **Refine.** MAP pillar fit is strong: trust and findability are explicit City Hall outcomes. Missing: **named owner** (DIT vs Communications vs 311 operations) and **KPI** (e.g. first-contact resolution proxy, widget sessions → correct outbound clicks). Political risk is **moderate**: any AI-adjacent resident-facing tool inherits NYC-style scrutiny if it ever drifts from “link to official page” into advice.

**Temporal:** **Appreciating** if EnerGov replacement and site IA improve; **stable** on the 311 wall (link-out remains the workaround).

---

### 2 — INNOVATOR (Expander)

**Reframe:** The product is not “a chatbot” but a **confidence-gated router**: refuse to answer when classification confidence is low; escalate to phone/311. Most leverage: **Pattern A** (curated intents + canonical URLs) matches the architecture already claimed (Claude classifies → KB returns links). **Provocative stretch:** invert to **proactive alerts** (sweeping, boil-water) — out of weekend scope but uses same trust model.

**Key insight:** Value = **what it refuses to synthesize**, not eloquence.

---

### 3 — FRONTLINER (Operator)

**311 / web staff:** **Maybe → Use** if the widget **reduces** “wrong ticket” volume and never pretends to open cases in AvePoint. **Risk:** residents blame 311 when the widget’s KB is stale. **Community workers:** **Maybe** — they rarely start on rva.gov; SMS parity (per INNOVATION_ANALYSIS divergent #2) matters for adoption outside the engaged segment.

**Failure modes:** Stale links after site migrations; classification sends user to wrong form once → trust collapse.

---

### 4 — CITIZEN (Recipient)

| Archetype | Verdict |
|-----------|---------|
| **Engaged** | Direct visibility if embedded on homepage; wants citations — **aligned** with “no hallucination” story |
| **Service User** | High value **if** answers are fast and links work; measures success in minutes saved |
| **Invisible** | **Gap:** widget + web assumes device and trust; voice and auto language help but **do not** replace phone-first or no-broadband users; translation quality varies by language (San Jose precedent) |

---

### 5 — ARCHITECT (System)

**Walls:** **311/AvePoint** — not hit for write path; **correct workaround** is deep links + phone. **Integration complexity:** **Low–Medium** (external host + embed script; optional Azure). **Data:** KB is **team-maintained** — not the same as “public API truth”; rva.gov crawl/RAG (Pattern B) raises **staleness and scope** risk per pillar **G4/D5**.

**AR 2.13:** Deployment needs **DIT governance**, public AI disclosure, data minimization for logs.

**Recommendation:** **Ship** a narrow MVP (classification + curated KB + blocklist); **Don’t** expand to generative answers without RAG + legal review.

---

### 6 — FUNDER (Resource)

**Model:** **Hybrid** — pilot can run on **low daily burn** (IDEA claims &lt;$10/day); **sustainability** hinges on **staff time** to refresh KB and monitor misroutes, not GPU cost. **Year 1:** cloud + APIs + light contractor or intern curation. **Year 3:** needs **operating owner** budget line or it becomes staleware.

**Sustainability risk:** **Medium** (content ops, not infra).

---

### 7 — ACTIVIST (Ecosystem)

**Scan:** Citibot/Denver, Ordinal/Decatur IN, NOLA Jazz, Durham procurement pilot — **Build** is justified **if** Richmond-specific and **open** where possible. **Decision tree:** **Adapt + Build** — adopt *patterns* (blocklist, escalation, citations), not a single vendor stack. **Ownership:** City should **endorse/embed** for trust; a **nonprofit or vendor** can operate hosting if contract and data rules are clean (contrast ChatNOLA / NOLA pause lesson).

## Fatal flaw test

| Criterion | Result | Notes |
|-----------|--------|--------|
| No identifiable owner? | **CONDITIONAL** | Natural homes: **DIT** (AR 2.13 + embed) + **Communications/Digital** (rva.gov). Must be **named** before “GO.” |
| Structural wall, no workaround? | **PASS** | No ticket API required if **linking** only. |
| Net negative for frontline? | **PASS** *if* scope disciplined | Becomes **FAIL** if KB drives systematic wrong-form submissions. |
| No funding beyond pilot? | **PASS** (infra); **MEDIUM** cliff on **content labor** | Not a fatal flaw if ops budget acknowledged. |
| Harms Invisible Resident? | **REFINE** | Web widget **under-serves** phone-only / low-trust users unless **311 phone** is always prominent and SMS/channel parity is planned. |

**Synthesis:** **No single fatal flaw** if owner + curation SLA + inclusion path are fixed. **Not** dead on arrival.

---

## Verdict

**REFINE → conditional GO**

**Ship when:**
1. **Owner + governance:** DIT sign-off under AR 2.13; disclosure copy for users; log/retention policy.  
2. **Scope lock:** Classification → **verified URLs only**; **blocklist** for legal/tax/emergency; low-confidence → human/311.  
3. **Freshness:** Process for KB updates when rva.gov changes; stale-link detection.  
4. **Equity:** Default path to **311 voice**; roadmap for **SMS** or partner channel if web-only is insufficient.

**Temporal tag:** **Appreciating** (better city web IA and future permitting stack could widen safe coverage); **Depreciating** if KB rots after launch.
